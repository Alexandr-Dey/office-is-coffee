# IDEAS.md — Office is Coffee

> Список идей для развития продукта. Одни идеи я собрал из анализа рынка и твоих запросов,
> другие — твои. Каждая идея оценена по **impact** (влияние на продукт) и **effort** (сложность реализации).
>
> Используй этот файл как меню: выбирай идеи, переноси их в TASKS.md когда готов делать.

---

## Как читать оценки

**Impact**: сколько пользы принесёт. ⭐ (мелочь) → ⭐⭐⭐⭐⭐ (меняет продукт)
**Effort**: сколько работы. 🔨 (час) → 🔨🔨🔨🔨🔨 (неделя+)
**Приоритет для запуска**: ✅ делать сейчас, 🟡 после запуска, ⚪ идеи на будущее

## 🎭 Раздел 1: Визуал и атмосфера сцены

### 1.1 Живые NPC с характерами ⭐⭐⭐⭐⭐ 🔨🔨🔨🔨 ✅
**Идея**: Не просто "пиксельные фигурки", а NPC с архетипами: студент с ноутбуком, бизнесмен с телефоном, мама с коляской, парочка, одинокий путник в капюшоне. У каждого архетипа — свои действия, позы, предметы.

**Почему важно**: сцена становится не декорацией, а маленьким спектаклем. Пользователь будет заходить в приложение, чтобы просто посмотреть что там происходит — это формирует привычку.

**Как реализовать**: 
- Массив архетипов с наборами анимаций и предметов
- При спавне NPC — случайный выбор архетипа
- Каждый архетип имеет свои "точки интереса" в сцене (студент → столик с розеткой, парочка → столик у окна)

---

### 1.2 Погода за окном ⭐⭐⭐⭐ 🔨🔨🔨 ✅
**Идея**: Окна в кофейне показывают актуальную погоду Алматы. Дождь — капли по стеклу и лужи на улице. Снег — снежинки и сугробы. Солнце — лучи света проникают внутрь, на полу световые пятна. Туман — размытый силуэт улицы.

**Почему важно**: это даёт "дыхание" сцене и связывает виртуальное с реальным. Пользователь видит, что в приложении "прямо сейчас" тоже идёт дождь — это магия.

**Как реализовать**: 
- OpenWeatherMap API (бесплатный тариф)
- Кешировать на 30 минут (cron Cloud Function раз в 30 мин → Firestore)
- В CoffeeScene — слой `<Weather />` с условным рендерингом

---

### 1.3 Время суток меняет освещение ⭐⭐⭐⭐ 🔨🔨 ✅
**Идея**: В 7 утра — розовый рассвет, окна оранжевые. В обед — яркое солнце, белый свет. Вечером — тёплый жёлтый свет ламп, окна тёмные. Ночью — только лампы, всё приглушено.

**Почему важно**: каждый заход в приложение будет визуально уникальным. Плюс это усиливает пасхалку "спящие баристы после 23:00".

**Как реализовать**:
- 4-6 пресетов освещения (dawn, morning, noon, afternoon, evening, night)
- Определяется по `new Date().getHours()`
- Меняет цвета стен, окон, пол (через CSS custom properties + Framer Motion)

---

### 1.4 Звуковое сопровождение ⭐⭐⭐⭐⭐ 🔨🔨🔨 🟡
**Идея**: Тихие эмбиент-звуки кофейни. Шум эспрессо-машины, перезвон чашек, приглушённый джаз, скрип двери когда кто-то входит. Все звуки — low volume, можно отключить в настройках.

**Почему важно**: звук — самый недооценённый фактор уюта. Без него сцена — молчаливый гифка. Со звуком — ты буквально "заходишь" в кофейню.

**Как реализовать**:
- Найти/купить loop-ы на freesound.org (CC0 лицензии)
- `<audio loop autoplay>` с fade-in/out
- Кнопка-переключатель mute в углу сцены
- Отключено по умолчанию (iOS не даёт autoplay без interaction)

**Риски**: на iPhone PWA работа со звуком ограничена, нужно тестировать.

---

### 1.5 Виталий и Аслан — живые характеры ⭐⭐⭐⭐ 🔨🔨🔨 ✅
**Идея**: Вместо абстрактных баристов — у них характеры. Виталий — педант, всегда протирает и раскладывает всё аккуратно. Аслан — шутник, делает смешные жесты, играет со стаканчиками (жонглирует когда никто не смотрит).

**Почему важно**: даёт продукту душу. У тебя ведь реальные баристы с именами — пусть они "живут" в приложении.

**Как реализовать**:
- Отдельные наборы idle-анимаций для каждого
- Виталий: wipe_counter, arrange_cups, check_machine, adjust_apron
- Аслан: juggle_cup, dance_move, check_phone, laugh, stretch
- Случайный выбор каждые 8-15 секунд

---

### 1.6 Пасхалки на особые даты ⭐⭐⭐ 🔨🔨 🟡
**Идея**: В день рождения кофейни (или города) — в сцене праздничное оформление: шарики, конфетти, баристы в колпачках. На Новый год — ёлка, гирлянды. На Наурыз — юрта и традиционные орнаменты.

**Почему важно**: даёт причины заходить в приложение в особые дни, создаёт чувство сопричастности.

**Как реализовать**:
- Массив `specialDates` в constants.ts
- Проверка текущей даты → добавление overlay-слоя в сцену

## 💳 Раздел 2: Оплата и лояльность

### 2.1 Kaspi Pay интеграция ⭐⭐⭐⭐⭐ 🔨🔨🔨🔨 🟡
**Идея**: Оплата напрямую через Kaspi. Нажал "Оплатить" → приложение Kaspi открывается с подготовленным платежом → вернулся обратно. Это стандарт в Казахстане.

**Почему важно**: без Kaspi в Казахстане нельзя делать платежи. Депозит — это обход, но Kaspi — настоящее решение.

**Как реализовать**:
- Kaspi API документация: https://pay.kaspi.kz/ (подключается через Merchant API)
- Нужен договор с Kaspi.kz (через юрлицо кофейни)
- Webhook приём подтверждения платежа → обновление заказа
- До получения доступа к API — работаем через депозит

**Блокер**: юридические вопросы с Kaspi, может занять недели.

---

### 2.2 Промокоды и акции ⭐⭐⭐⭐ 🔨🔨🔨 🟡
**Идея**: Система промокодов (процент скидки, фиксированная сумма, бесплатный напиток). CEO создаёт через /ceo, клиент вводит при оформлении.

**Примеры акций**: "Happy hour 14:00-16:00 — 20% на всё", "Второй кофе за 50%", "День рождения — бесплатный раф".

**Почему важно**: инструмент маркетинга и возврата клиентов. Без промокодов сложно запускать рекламные кампании.

**Как реализовать**:
- Коллекция `promo_codes/{code}` с полями (type, value, maxUses, expiresAt, usedBy[])
- Cloud Function `applyPromoCode` для атомарной валидации и применения
- В /ceo форма создания, в /order поле ввода

---

### 2.3 Реферальная программа ⭐⭐⭐⭐ 🔨🔨🔨 🟡
**Идея**: Пригласи друга — получи бесплатный кофе. Друг регистрируется по твоей ссылке, делает первый заказ — вам обоим начисляется что-то.

**Почему важно**: органический рост с минимальными затратами. Особенно эффективно среди друзей и тестеров.

**Как реализовать**:
- Каждый юзер получает уникальный `referralCode` (6 символов)
- Shareable ссылка: `https://office-is-coffee.vercel.app/?ref=ABC123`
- При регистрации с `?ref=` — бонусы обоим после первого заказа приглашённого
- Виджет "Поделиться с другом" в профиле

---

### 2.4 Подписка "Безлимит" ⭐⭐⭐⭐⭐ 🔨🔨🔨🔨 ⚪
**Идея**: Месячная подписка (например, 15 000₸) = любое количество кофе в день (1 или 2). Как у сервиса "Coffee Cloud" или Pret A Manger Coffee Subscription.

**Почему важно**: стабильный доход кофейне, выгода клиенту, залочивание на бренд. Это работает — Pret продаёт тысячи таких подписок.

**Как реализовать**:
- Коллекция `subscriptions/{uid}` с полями (active, plan, expiresAt, dailyLimit, usedToday)
- Cloud Function каждое утро сбрасывает `usedToday`
- При заказе — проверка подписки, если активна — total = 0
- Оплата подписки — отдельный flow (Kaspi)

**Блокер**: нужна Kaspi интеграция.

---

### 2.5 Подарочные сертификаты ⭐⭐⭐ 🔨🔨🔨 🟡
**Идея**: Купи сертификат на сумму, отправь другу ссылкой. Друг переходит, получает QR, приходит в кофейню и использует.

**Почему важно**: низкий эффорт, но работает как маркетинговый инструмент для сарафана.

**Как реализовать**:
- Аналогично промокодам, но с денежной стоимостью и QR для передачи
- Можно в первой версии сделать вручную (CEO через /ceo создаёт сертификат, отправляет код другу)

## 📱 Раздел 3: UX и функционал

### 3.1 Заказ заранее (pre-order) ⭐⭐⭐⭐⭐ 🔨🔨🔨 ✅
**Идея**: Клиент заказывает кофе "к 9:15 утра". Утром приходит — кофе уже готов, без ожидания. Бариста получает уведомление заранее (в 9:10).

**Почему важно**: **ключевая фича для делового района**. Утренние офисные клиенты — самая ценная аудитория. Они готовы платить за экономию 5 минут.

**Как реализовать**:
- В /order добавить toggle "Заказать заранее" → time picker
- Поле `orders/{id}.scheduledFor: timestamp`
- Cloud Function каждые 5 мин проверяет scheduled заказы → за 5 мин до времени создаёт настоящий заказ и шлёт пуш баристе
- В /admin отдельная вкладка "Предзаказы"

---

### 3.2 Геолокация "Я подхожу" ⭐⭐⭐⭐ 🔨🔨 ✅
**Идея**: Когда клиент входит в радиус 200м от кофейни — автоматически шлётся пуш баристе "Клиент Саша в 200м, готовь заказ". Бариста успевает всё подготовить.

**Почему важно**: клиент приходит — кофе уже почти готов. Меньше ожидания, больше счастья.

**Как реализовать**:
- Service Worker + background geolocation (через `navigator.geolocation.watchPosition`)
- Когда заказ в статусе `new` или `pending` и клиент в радиусе — триггер
- iOS ограничивает background geolocation, возможно придётся ограничиться foreground режимом

---

### 3.3 Избранное и "обычный заказ" ⭐⭐⭐⭐ 🔨🔨 ✅
**Идея**: Клиент отмечает свой любимый напиток → он всегда наверху меню с кнопкой "Мой обычный — 1 тап заказать". Большинство клиентов заказывают одно и то же каждый день.

**Почему важно**: заказ за 1 тап — фундамент ретеншна. Starbucks и Pret так работают.

**Как реализовать**:
- Поле `users/{uid}.favoriteItem` уже есть в схеме
- Автоопределение: если 5+ раз заказывал одно — предложить сохранить в избранное
- Кнопка "Мой обычный" в hero-зоне /menu

---

### 3.4 Tip jar — чаевые баристе ⭐⭐⭐ 🔨🔨 🟡
**Идея**: После получения заказа клиент может оставить чаевые конкретному баристе. Сумма идёт в `barista_bonuses`, CEO выплачивает вместе с основными бонусами.

**Почему важно**: мотивация баристам, новый источник их дохода, культурно правильно.

**Как реализовать**:
- На экране "Оцени заказ" добавить "Оставить чаевые: 100 / 200 / 500 / свой вариант"
- Cloud Function списывает с депозита → добавляет в бонусы конкретному баристе

---

### 3.5 Комментарий к заказу ⭐⭐ 🔨 ✅
**Идея**: Поле "Комментарий баристе" при оформлении (например, "очень горячий", "без сахара", "для Марии"). Поле `comment` уже в схеме, но не знаю — используется ли в UI.

**Почему важно**: персонализация, понимание клиента.

**Как реализовать**:
- В /order добавить textarea "Комментарий (необязательно)"
- Показывать в карточке заказа в /admin крупным текстом

---

### 3.6 Режим "Я в пути" ⭐⭐⭐ 🔨 ✅
**Идея**: Кнопка на экране ожидания заказа — "Я в пути, выхожу с работы". Баристе приходит уведомление "Клиент идёт, ~5 минут".

**Почему важно**: снижает риск того, что кофе остынет, и помогает баристе планировать.

**Как реализовать**:
- Кнопка на /order/[id]
- Cloud Function отправляет пуш конкретному баристе (по `baristaid`)

## 🎮 Раздел 4: Геймификация

### 4.1 Система ачивок ⭐⭐⭐⭐ 🔨🔨🔨 🟡
**Идея**: Бейджики за достижения: "Первый заказ", "Неделя подряд", "Попробовал 10 разных напитков", "Ночной совёнок" (заказ после 22:00), "Ранняя пташка" (заказ до 8:00), "Мастер рафа" (10 рафов), "Последний заказ года".

**Почему важно**: геймификация — одна из самых эффективных механик ретеншна. Starbucks делают это блестяще.

**Как реализовать**:
- Коллекция `achievements/{id}` с определениями
- При каждом заказе — Cloud Function проверяет условия и выдаёт бейджики
- Экран /profile показывает коллекцию

---

### 4.2 Дневные челленджи ⭐⭐⭐ 🔨🔨🔨 🟡
**Идея**: "Сегодняшний челлендж — попробуй что-то новое и получи +20 монет". Меняется каждый день.

**Почему важно**: причина зайти в приложение каждый день.

**Как реализовать**:
- Cloud Function раз в день генерирует челлендж из пула
- При выполнении — начисление бонуса

---

### 4.3 Монеты → реальные награды ⭐⭐⭐ 🔨🔨 ✅
**Идея**: У тебя уже есть /coins экран, но непонятно — на что монеты тратятся. Сделать каталог: "100 монет = скидка 500₸", "500 монет = бесплатный круассан".

**Почему важно**: без смысла монеты — бесполезный прогресс-бар. Со смыслом — мощный ретеншн.

**Как реализовать**:
- Коллекция `rewards/{id}` с ценой в монетах
- Обмен через Cloud Function (проверка баланса, уменьшение, генерация QR для баристы)

---

### 4.4 Leaderboard среди друзей ⭐⭐ 🔨🔨🔨 ⚪
**Идея**: Видеть чей стрик длиннее среди твоих друзей.

**Почему важно**: социальное давление работает для ретеншна.

**Как реализовать**:
- Друзья через ссылки-приглашения (как в п.2.3)
- Агрегация топа раз в день через Cloud Function

**Приватность**: осторожно, нужен opt-in.

## 🏪 Раздел 5: Операционная эффективность

### 5.1 Статистика для CEO ⭐⭐⭐⭐⭐ 🔨🔨🔨 ✅
**Идея**: Расширенный дашборд — не только выплаты баристам, но и: выручка по дням/неделям/месяцам, топ-напитки, среднее время приготовления, пиковые часы, NPS (из рейтингов), конверсия онбординга.

**Почему важно**: без аналитики невозможно управлять бизнесом. Это твой главный инструмент как владельца.

**Как реализовать**:
- Cloud Function раз в час агрегирует данные → `analytics/{date}` документ
- Recharts для графиков
- Экспорт в CSV

---

### 5.2 Стоп-лист с автоматикой ⭐⭐⭐ 🔨🔨 ✅
**Идея**: Бариста указывает "осталось 5 порций облепихового чая" → после 5 заказов автоматически в стоп-лист. Утром CEO видит уведомление "что закончилось вчера".

**Почему важно**: избавляет баристу от ручного обновления стоп-листа.

---

### 5.3 Управление расписанием баристов ⭐⭐⭐ 🔨🔨🔨 🟡
**Идея**: CEO составляет расписание смен, бариста видит свою смену. Только бариста на смене получает уведомления о заказах.

**Почему важно**: два баристы не должны оба получать пуш о каждом заказе, если работает один.

**Как реализовать**:
- Коллекция `shifts/{date}` с массивом баристов на смене
- Cloud Function фильтрует получателей пушей по shifts

---

### 5.4 Управление меню через /ceo ⭐⭐⭐⭐ 🔨🔨🔨 ✅
**Идея**: После миграции меню в Firestore — CEO через /ceo может редактировать напитки (цены, описания, теги, доступность). Без перекомпиляции.

**Почему важно**: кофейня периодически меняет цены и линейку — без этого каждый раз нужна помощь разработчика.

**Как реализовать**:
- Вкладка "Меню" в /ceo
- CRUD операции над `menu_items`
- Только для роли `ceo`

---

## 🔐 Раздел 6: Безопасность и стабильность

### 6.1 Rate limiting ⭐⭐⭐ 🔨🔨 🟡
**Идея**: Ограничение количества заказов в минуту на одного юзера (например, 3 заказа/мин) → защита от ошибочных дабл-кликов и злоумышленников.

**Как реализовать**:
- Поле `users/{uid}.orderRateLimit: { count, windowStart }`
- Cloud Function проверяет при создании заказа

---

### 6.2 Мониторинг ошибок ⭐⭐⭐⭐ 🔨 ✅
**Идея**: Подключить Sentry полностью (DSN настроить, тестовая ошибка). Без этого мы не узнаем о багах у пользователей.

**Почему важно**: тестеры жалуются обычно лениво — Sentry покажет баги, о которых никто не написал.

---

### 6.3 Бэкапы Firestore ⭐⭐⭐⭐ 🔨🔨 🟡
**Идея**: Автоматический экспорт Firestore в GCS раз в день. Настраивается в Firebase Console → Extensions.

**Почему важно**: один сбой — и вся база потеряна. Бэкапы — обязательное требование продакшна.

---

## 🎁 Раздел 7: Социальное и маркетинг

### 7.1 Instagram-friendly share ⭐⭐⭐ 🔨🔨 🟡
**Идея**: Красивая карточка для стори с твоим заказом ("Мой любимый раф в Love is Coffee #loveiscoffee"). Кнопка "Поделиться в инстаграм".

**Как реализовать**:
- Canvas API для генерации картинки
- Web Share API для системного шаринга

---

### 7.2 Интеграция с Apple Wallet / Google Pay ⭐⭐⭐ 🔨🔨🔨🔨 ⚪
**Идея**: Карта лояльности добавляется в Apple Wallet / Google Pay. Показываешь на кассе.

**Почему важно**: менее надо доставать приложение, быстрее чекаут.

**Блокер**: сложная интеграция, требует Apple Developer аккаунт ($99/год).

### 7.3 Push-уведомления о новом напитке ⭐⭐⭐ 🔨 ✅

## 📊 Раздел 8: Технические улучшения

### 8.1 Офлайн-режим ⭐⭐⭐⭐ 🔨🔨🔨🔨 🟡
**Идея**: Приложение работает без интернета. Меню кешируется, можно посмотреть. Заказы ставятся в очередь и отправляются при восстановлении связи.

**Почему важно**: в кофейне часто плохой wi-fi, мобильная сеть может "проседать". Без офлайна UX страдает.

**Как реализовать**:
- Service Worker кеширует статику и Firestore данные
- IndexedDB для очереди заказов
- UI-индикатор "офлайн"

---

### 8.2 Оптимизация bundle size ⭐⭐⭐ 🔨🔨 ✅
**Идея**: Анализ `next build` → найти большие библиотеки → заменить/lazy load. Цель — First Contentful Paint < 1.5s на 3G.

**Как реализовать**:
- `@next/bundle-analyzer` для анализа
- Code splitting по роутам
- Lazy load для avatar/page.tsx (883 строки — он грузится всегда?)

---

### 8.3 Image optimization ⭐⭐ 🔨 ✅
**Идея**: Все картинки через `next/image`. Вебp формат. Оптимизированные иконки PWA.

**Как реализовать**: стандартный процесс Next.js.

---

### 8.4 Server-side rendering для меню ⭐⭐⭐ 🔨🔨 🟡
**Идея**: Сейчас меню рендерится на клиенте → первый экран пустой пока не загрузятся данные. SSR сделает первую картинку мгновенной.

**Как реализовать**:
- `menu/page.tsx` → async Server Component
- Firebase Admin SDK на сервере для чтения `menu_items`
