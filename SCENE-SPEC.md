# SCENE-SPEC.md — CoffeeScene Specification

> Детальная спецификация главной фичи приложения — интерактивной сцены кофейни.
> Это **самая важная часть продукта**. Пользователи должны заходить в приложение,
> чтобы посмотреть что там происходит — не только заказать кофе.
>
> Claude Code должен перечитывать этот документ каждый раз перед работой над CoffeeScene.tsx.
>
> Версия: 1.0 (2026-04-07, с нуля, не путать с текущей реализацией)

---

## 1. Визия

### Что мы делаем

**Живой pixel-art мирок**, в котором:
- Реальные баристы (Виталий и Аслан) живут своей жизнью
- Клиенты приходят, заказывают, ждут, уходят
- Всё происходит в реальном времени
- Каждый заход в приложение — немного другой
- Видны детали: пар от кофемашины, блики от ламп, капли дождя в окнах

### Что это НЕ

- ❌ НЕ просто анимированный баннер сверху экрана
- ❌ НЕ статичная картинка с одной анимацией
- ❌ НЕ минималистичный flat design
- ❌ НЕ символическая иллюстрация "что здесь кофейня"

### Референсы

- **Coffee Talk** (игра) — атмосферные ночные сцены кофейни
- **Stardew Valley** — детализированный pixel-art с характером
- **Tiny Tower** (игра) — маленький мир с живыми NPC
- **Habitica** — геймификация через пиксельные персонажи
- **Monument Valley** — внимание к деталям и магия

## 2. Техническая архитектура

### 2.1 Выбор технологии: SVG + sprite sheets

**Основа — SVG**, потому что:
- Масштабируется без потерь качества
- React-friendly (каждый элемент — DOM-нода)
- Framer Motion работает нативно
- Accessibility через `<title>` и `<desc>`

**Персонажи и сложные объекты — PNG sprite sheets**, потому что:
- Pixel-art персонажей рисовать в коде через `<rect>` — мучение
- Готовые спрайты можно подготовить в Aseprite или сгенерировать через AI
- Анимация через `background-position` (простая и производительная)

### 2.2 Гибридный подход

```
CoffeeScene (root SVG)
├── BackgroundLayer (SVG rects для стен/пола)
├── FurnitureLayer (SVG rects для стойки, столов)
├── ObjectsLayer (SVG + sprite PNG для кофемашины, микроволновки)
├── SpritesLayer (PNG sprites через <image> или <foreignObject>)
│   ├── VitaliySprite (256x256 sprite sheet, 8 кадров анимации)
│   ├── AslanSprite (256x256 sprite sheet, 8 кадров анимации)
│   ├── NpcSprites[] (procedurally composed from body parts)
│   └── ...
└── EffectsLayer (SVG: пар, капли, блики — анимированные)
```

### 2.4 viewBox и координаты

```
viewBox="0 0 800 600"
```

Почему 800x600 (а не 390x220 как сейчас):
- Больше рабочего пространства для детализации
- Соотношение 4:3 близко к тому как выглядит реальная сцена
- На мобильном масштабируется пропорционально
- Лучше для pixel-art (больше места для деталей)

**Зоны сцены** (координаты в viewBox):

```
┌─────────────────────────────────────────┐ (0, 0)
│                                         │
│  OKNA (0-800, 0-150)                    │  ← окна и погода
│                                         │

│                                         │

│       │  (objects on it) │              │
```

### 2.5 Z-order (порядок отрисовки)

От дальнего к ближнему:

1. **Фон окна** (небо, погода)
2. **Стены** (красная + светлая)
3. **Окна** (рамки, подоконники)
4. **Меню-борды и постеры на стенах**
5. **Полки с товарами**
6. **Объекты ЗА стойкой** (холодильник, сифоны)
7. **Баристы** (рисуются ПЕРЕД стойкой ТОЛЬКО головой и плечами, ноги скрыты стойкой)
8. **Сама стойка** (рисуется поверх ног баристов, но под их руками на стойке)
9. **Объекты НА стойке** (кофемашина, микроволновка, стаканчики)
10. **Руки баристов работающие на стойке** (нарисованы поверх объектов если нужно)
11. **Пар от кофемашины**
12. **NPC клиенты** (в зависимости от y-координаты: ближние — впереди)
13. **Пол** (не закрывает ничего, просто цвет снизу)
14. **Эффекты освещения** (overlay с opacity)
15. **Облачко статуса заказа** (над головой баристы)

## 4. Персонажи — баристы

### 4.1 Виталий (Vitaliy)

**Характер**: педант, перфекционист, следит за порядком. Всегда что-то протирает, поправляет, раскладывает. Интроверт, но дружелюбный.

**Внешность**:
- Кожа: светлая #e8b88a
- Волосы: тёмно-каштановые #2c1810, средней длины, немного взлохмаченные
- Глаза: зелёные
- Фартук: синий #2980b9 с логотипом Love is Coffee
- Футболка: белая
- Штаны: тёмно-синие
- Особая примета: часы на руке

**Положение в сцене**: слева, обычно у кофемашины или протирает стойку.

**Idle действия** (случайные, 8-15 секунд каждое):
1. `wipe_counter` — протирает стойку тряпкой (движение руки влево-вправо)
2. `check_machine` — нагибается к кофемашине, поправляет что-то
3. `adjust_apron` — подтягивает лямки фартука
4. `stretch_back` — потягивается назад, держась за поясницу
5. `polish_cup` — полирует кружку тряпкой
6. `check_phone` — достаёт телефон, смотрит, убирает
7. `organize_cups` — переставляет стопку стаканчиков
8. `look_at_clock` — смотрит на часы на руке
9. `tap_rhythm` — постукивает пальцами в ритм воображаемой музыки
10. `smell_beans` — нюхает зёрна из контейнера, довольный
11. `drink_water` — пьёт воду из своего стакана
12. `wave_at_camera` — смотрит "в камеру" (на пользователя) и машет (редко, 1%)

### 4.2 Аслан (Aslan)

**Характер**: шутник, экстраверт, энергичный. Любит жонглировать, танцевать, делать смешные жесты. Душа компании.

**Внешность**:
- Кожа: смуглая #d4a574
- Волосы: чёрные #1a1a1a, короткие
- Глаза: карие
- Фартук: зелёный #27ae60 с логотипом
- Футболка: белая
- Штаны: тёмно-синие
- Особая примета: браслет из бусин

**Положение в сцене**: справа, обычно у зоны выдачи или играет со стаканчиками.

**Idle действия**:
1. `juggle_cup` — жонглирует одним стаканчиком (бросает-ловит)
2. `dance_move` — короткий танцевальный шаг
3. `check_phone` — листает телефон с улыбкой
4. `laugh` — смеётся (над чем-то или над Виталием)
5. `stretch_arms` — разводит руки в стороны
6. `talk_to_vitaliy` — поворачивается к Виталию, что-то говорит (Виталий в этот момент кивает)
7. `wipe_cup` — протирает стаканчик с преувеличенным вниманием
8. `pose` — встаёт в "модельную позу" (руки на бёдрах)
9. `air_drums` — барабанит по стойке пальцами как по барабанам
10. `peek_at_oven` — заглядывает в микроволновку
11. `write_on_cup` — пишет маркером на стаканчике
12. `finger_guns` — делает жест "пистолеты" пальцами в камеру (редко)

### 4.3 Взаимодействия между баристами

Иногда (каждые 30-60 секунд) запускается **совместное действие**:
1. `aslan_tells_joke` — Аслан что-то говорит, Виталий слушает, оба смеются
2. `high_five` — бьют пятюней (если оба свободны)
3. `rock_paper_scissors` — играют в "камень-ножницы-бумага"
4. `vitaliy_teaches_aslan` — Виталий что-то показывает Аслану на кофемашине
5. `cheers_cups` — чокаются стаканчиками

Эти действия делают сцену "одушевлённой" — пользователь видит, что это живые люди, а не болванчики.

### 4.4 Реакции на статус заказа

Когда у пользователя есть активный заказ, приоритет idle-действий падает — баристы работают над заказом.

**Статус `pending`** (заказ только создан):
- Оба поворачиваются к "экрану" (смотрят туда, где у реальной кассы экран заказов)
- Виталий кивает, принимая заказ
- Над сценой появляется облачко "Ваш заказ принят"
- Idle-действия приостановлены на 3 секунды

**Статус `accepted`** (бариста начал делать):
- Виталий идёт к кофемашине (анимация ходьбы 2 шага)
- Начинается "работа у машины" анимация: руки двигаются, ручки крутятся
- Пар от машины усиливается (opacity 0.3 → 0.7, амплитуда больше)
- Аслан берёт стаканчик, пишет на нём имя клиента маркером
- Это состояние длится пока статус `accepted`

**Статус `ready`** (готов):
- Виталий отходит от машины
- Аслан ставит готовый стаканчик на зону выдачи
- Аслан машет рукой "забирай!" (анимация 3 взмахов)
- Появляется облачко "Забирай у Аслана ☕"
- Аватар пользователя в сцене (если он был в очереди) идёт к стойке

**Статус `paid`**:
- Аватар пользователя берёт стаканчик со стойки
- Идёт к двери
- Выходит (fade out на двери)
- Баристы возвращаются к idle (с небольшой паузой 2 секунды)

### 4.5 Грустный Виталий

**Условие**: `streak === 0 && daysSinceLastOrder >= 2`

**Анимация**:
- Стоит в углу у кассы
- Голова опущена
- Рот перевёрнут (грустный)
- Смотрит на дверь (ждёт когда кто-то вернётся)
- Иногда вздыхает (плечи поднимаются-опускаются)
- Над головой облачко мысли с "?"

**Это мощный эмоциональный триггер** — пользователь увидит что его баристе грустно и вернётся за кофе.

### 4.6 Пасхалки баристов

- **8× тап на Виталия** за 10 секунд → злится (красное лицо), разворачивается, бросает фартук, уходит за левый край сцены. Через 30 секунд возвращается с улыбкой, поднимает фартук.
- **5× тап на Аслана** за 10 секунд → пытается сделать сальто. Прыжок, вращение, падение (смешно), встаёт, отряхивается, смущённо смотрит.
- **Тап на кофемашину** → пар выстреливает со свистом (`steam_burst` анимация), Виталий пугается и отскакивает.
- **Тап на микроволновку** → открывается, внутри пусто, Аслан разочарованно закрывает.
- **Тап на стаканчики** → они слегка покачиваются, верхний падает, Аслан ловит.

## 5. NPC — клиенты (процедурная генерация)

### 5.1 Общая концепция

В сцене постоянно находятся клиенты — **реальные** (у которых есть активный заказ в приложении) и **фейковые NPC** (для атмосферы).

**Распределение**:
- Реальные клиенты: 1-3 (сколько активных заказов в системе)
- Фейковые NPC: 2-5 (случайное число, меняется каждые 2-5 минут)

### 5.2 Процедурная генерация (proc-gen)

Каждый NPC генерируется из компонентов:

```ts
interface NpcAppearance {
  seed: number;              // для стабильной генерации
  bodyColor: string;         // оттенок кожи: 4 варианта
  hairStyle: number;         // 0-7 (8 причёсок)
  hairColor: string;         // 6 цветов
  shirtStyle: number;        // 0-5 (6 стилей одежды)
  shirtColor: string;        // 12 цветов
  pantsColor: string;        // 4 цвета (тёмные)
  accessory: string | null;  // 'backpack', 'phone', 'book', 'tote-bag', null
  height: 'short' | 'medium' | 'tall';  // +/- 4 пикселя
}
```

**Генерация из seed**:
```ts
function generateNpc(seed: number): NpcAppearance {
  const rng = seededRandom(seed);
  
  return {
    seed,
    bodyColor: SKIN_TONES[rng.int(0, SKIN_TONES.length - 1)],
    hairStyle: rng.int(0, 7),
    hairColor: HAIR_COLORS[rng.int(0, HAIR_COLORS.length - 1)],
    shirtStyle: rng.int(0, 5),
    shirtColor: SHIRT_COLORS[rng.int(0, SHIRT_COLORS.length - 1)],
    pantsColor: PANTS_COLORS[rng.int(0, PANTS_COLORS.length - 1)],
    accessory: rng.pick(ACCESSORIES),
    height: rng.pick(['short', 'medium', 'tall']),
  };
}
```

**Важно**: seed сохраняется. Один и тот же NPC (реальный клиент) всегда выглядит одинаково. У настоящего юзера seed = хеш от его `uid`.

### 5.3 Архетипы (для фейковых NPC)

Архетип определяет поведение + косметику:

#### Архетип 1: Студент
- Одежда: худи или футболка ярких цветов
- Аксессуар: рюкзак или наушники
- Поведение: идёт к столику с ноутбуком, садится, печатает
- Заказ: обычно латте или раф

#### Архетип 2: Бизнесмен
- Одежда: рубашка или пиджак нейтральных цветов
- Аксессуар: портфель или телефон
- Поведение: стоит в очереди, проверяет телефон, берёт кофе навынос
- Заказ: эспрессо или американо

#### Архетип 3: Мама с ребёнком
- Одежда: комфортная, пастельные цвета
- Аксессуар: коляска или за руку держит маленького NPC
- Поведение: садится за большой столик, заказывает что-то неторопливо
- Заказ: капучино или чай

#### Архетип 4: Парочка
- Два NPC, идут близко друг к другу
- Одежда: стильная
- Поведение: садятся за столик у окна, болтают, смеются
- Заказ: два авторских напитка

#### Архетип 5: Спортсмен
- Одежда: спортивная
- Аксессуар: бутылка воды
- Поведение: забегает, быстро берёт и уходит
- Заказ: фреш или смузи

#### Архетип 6: Художник/фрилансер
- Одежда: винтажная, необычная
- Аксессуар: блокнот или iPad с стилусом
- Поведение: садится в угол, что-то рисует, сидит долго
- Заказ: матча или сезонный

#### Архетип 7: Турист
- Одежда: практичная, с рюкзаком
- Аксессуар: фотоаппарат
- Поведение: оглядывается, фотографирует интерьер
- Заказ: что-то известное (капучино)

### 5.4 Движение NPC (pathfinding)

NPC не телепортируются — они **ходят** по сцене. Движение через Framer Motion `animate`:

**Точки интереса** (waypoints):
```ts
const WAYPOINTS = {
  door: { x: 740, y: 480 },           // вход/выход
  queue1: { x: 380, y: 450 },         // первое место в очереди
  queue2: { x: 420, y: 450 },         // второе место в очереди
  queue3: { x: 460, y: 450 },         // третье место в очереди
  counter: { x: 340, y: 440 },        // у кассы
  pickup: { x: 460, y: 440 },         // зона выдачи
  table1: { x: 100, y: 490 },         // столик слева
  table2: { x: 680, y: 490 },         // столик справа
  exit: { x: 750, y: 490 },           // у двери перед выходом
};
```

**Жизненный цикл NPC**:
```
1. Spawn у двери (door, fade in)
2. Move → queue (встаёт в очередь)
3. Wait → (5-15 секунд, сдвигается вперёд когда очередь двигается)
4. Move → counter (подходит к кассе)
5. Interact with Aslan (3 секунды)
6. Move → waiting area (отходит)
7. Wait → pickup (2-5 секунд)
8. Move → pickup (забирает готовый кофе)
9. Decision: навынос или посидеть?
   a) Навынос: Move → exit → despawn
   b) Посидеть: Move → tableX → sit (30-120 секунд) → Move → exit → despawn
```

**Анимация движения**:
```tsx
<motion.g
  animate={{
    x: currentWaypoint.x,
    y: currentWaypoint.y,
  }}
  transition={{
    duration: distance / WALK_SPEED,  // скорость ~30px/сек
    ease: 'linear',
  }}
>
  <NpcSprite appearance={npc} action={isWalking ? 'walk' : 'idle'} />
</motion.g>
```

### 5.5 Реальные клиенты

Когда у реального пользователя есть активный заказ, его **персонаж** появляется в сцене:

**Статус `new`/`pending`**:
- Spawn у двери
- Идёт к очереди
- Встаёт в хвост

**Статус `accepted`**:
- Стоит в зоне ожидания
- Иногда проверяет телефон (свою игрушечную версию в пиксель-арте)
- Ждёт

**Статус `ready`**:
- Идёт к зоне выдачи
- Аслан машет → персонаж подходит
- Принимает стаканчик (анимация "взять")

**Статус `paid`**:
- Идёт к двери
- Выходит (fade out)

**Важно**: аватар генерируется из `users/{uid}.sceneAvatar`, сохраняется при регистрации. Один и тот же юзер всегда выглядит одинаково.

### 5.6 Взаимодействия между NPC

Иногда (10% шанс) два NPC делают одно из:
1. **Здороваются** — машут друг другу
2. **Сталкиваются** — один извиняется (emote "!")
3. **Болтают в очереди** — поворачиваются друг к другу
4. **Берут столик вместе** — двое незнакомых вынужденно сели за один столик

Эти мелочи — то, что делает сцену "живой" на уровне, который замечаешь подсознательно.

## 6. Связь с данными (real-time)

### 6.1 Что читает сцена

```tsx
// CoffeeScene.tsx props
interface CoffeeSceneProps {
  userUid: string;                     // текущий юзер
  userAvatar: NpcAppearance | null;    // для отрисовки его персонажа

  allActiveOrders: Order[];            // все активные заказы в системе
  cafeStatus: CafeStatus;              // открыта/закрыта
  streakDays: number;                  // стрик юзера
  lastOrderDate: string | null;        // для грустного Виталия
  weatherCondition: WeatherCondition;  // погода (из кеша)
}
```

### 6.2 Firestore listeners

В `/menu/page.tsx`:

```tsx
// Listener 1: мой активный заказ
useEffect(() => {
  if (!uid) return;
  const q = query(
    collection(db, 'orders'),
    where('userId', '==', uid),
    where('status', 'in', ['pending', 'accepted', 'ready']),
    orderBy('createdAt', 'desc'),
    limit(1)
  );
  return onSnapshot(q, (snap) => {
    setActiveOrder(snap.docs[0]?.data() ?? null);
  });
}, [uid]);

// Listener 2: все активные заказы (для отображения других клиентов в сцене)
useEffect(() => {
  const q = query(
    collection(db, 'orders'),
    where('status', 'in', ['pending', 'accepted', 'ready']),
    orderBy('createdAt', 'desc'),
    limit(5)
  );
  return onSnapshot(q, (snap) => {
    setAllActiveOrders(snap.docs.map(d => d.data()));
  });
}, []);

// Listener 3: статус кафе
useEffect(() => {
  return onSnapshot(doc(db, 'cafe_status', 'aksay_main'), (snap) => {
    setCafeStatus(snap.data());
  });
}, []);
```

### 6.3 Реакция сцены на изменения

**При появлении нового `activeOrder`**:
- Анимация "мой персонаж входит в сцену"
- Баристы реагируют (поворачиваются)

**При изменении `activeOrder.status`**:
- Плавный переход баристов из одного состояния в другое
- Персонаж юзера двигается соответственно

**При открытии кафе** (`cafeStatus.isOpen: false → true`):
- Сцена "просыпается"
- Баристы приходят в сцену (анимация "входят из-за кадра")
- Начинают работать

**При закрытии кафе**:
- Сцена затемняется
- Баристы уходят за кадр
- Остаётся пустая сцена

## 7. Время суток и погода

### 7.1 Система освещения

```ts
function getTimeOfDayPreset(date: Date): LightingPreset {
  const hour = date.getHours();
  
  if (hour < 6) return 'night';
  if (hour < 8) return 'dawn';
  if (hour < 11) return 'morning';
  if (hour < 15) return 'noon';
  if (hour < 18) return 'afternoon';
  if (hour < 21) return 'evening';
  return 'night';
}
```

**Пресеты** (меняют overlay и цвета):

| Пресет | Характеристика | Overlay |
|---|---|---|
| `dawn` | Розовый рассвет | `rgba(255, 180, 120, 0.15)` |
| `morning` | Свежий яркий | `rgba(255, 240, 200, 0.05)` |
| `noon` | Белый яркий | нет |
| `afternoon` | Тёплый жёлтый | `rgba(255, 220, 150, 0.08)` |
| `evening` | Оранжевые лампы | `rgba(255, 180, 100, 0.15)` |
| `night` | Тёплый тёмный | `rgba(40, 30, 80, 0.25)` |

Overlay — это большой `<rect>` на всю сцену с `mix-blend-mode: multiply` или `overlay`.

### 7.2 Погода (через API)

Используем OpenWeatherMap API (бесплатный план):

```ts
// Cloud Function, запускается каждые 30 минут
export const updateWeather = onSchedule('every 30 minutes', async () => {
  const response = await fetch(
    `https://api.openweathermap.org/data/2.5/weather?lat=43.2220&lon=76.8512&appid=${API_KEY}`
  );
  const data = await response.json();
  
  await db.collection('cache').doc('weather').set({
    condition: data.weather[0].main,  // 'Clear', 'Rain', 'Snow', 'Clouds', etc.
    temp: data.main.temp - 273.15,
    updatedAt: FieldValue.serverTimestamp(),
  });
});
```

Клиент читает `cache/weather` через `onSnapshot` и отображает в окнах сцены.

**Условия погоды** (что рисуется в окнах):

| Condition | Визуал |
|---|---|
| `Clear` | Голубое небо, солнце (в окнах лучи света на полу) |
| `Clouds` | Серое небо, облака |
| `Rain` | Капли на стекле (анимация), лужи на улице, потемневшее небо |
| `Snow` | Снежинки (анимация), белая улица, сугробы |
| `Mist`/`Fog` | Размытое окно (blur filter), серый туман |
| `Thunderstorm` | Молнии (редкие вспышки), тёмное небо, сильный дождь |

## 8. Эффекты

### 8.1 Пар от кофемашины

```tsx
<EffectsLayer>
  {[0, 1, 2].map(i => (
    <motion.ellipse
      key={i}
      cx={310 + i * 5}
      cy={280}
      rx={8}
      ry={12}
      fill="white"
      initial={{ opacity: 0, y: 0, ry: 12 }}

        opacity: [0, 0.5, 0],
        y: -50,
        ry: 20,
      }}
      transition={{
        duration: 3,
        delay: i * 0.5,
        repeat: Infinity,
        ease: 'easeOut',

  ))}
</EffectsLayer>
```

Интенсивность пара зависит от статуса заказа:
- idle/pending: 3 облачка, медленно
- accepted: 6 облачков, быстрее
- ready: 3 облачка, медленно

### 8.2 Капли дождя (если погода дождь)

```tsx
{Array.from({ length: 30 }).map((_, i) => (
  <motion.line
    key={i}

    y1={-10}
    x2={Math.random() * 800 - 10}
    y2={10}
    stroke="rgba(150, 200, 255, 0.5)"
    strokeWidth={1}
    initial={{ y: -10 }}
    animate={{ y: 600 }}
    transition={{
      duration: 0.8,
      delay: Math.random() * 3,
      repeat: Infinity,
      ease: 'linear',
    }}
  />
))}
```

### 8.3 Частицы в лучах света

Когда `preset === 'morning'` или `preset === 'afternoon'`, в лучах солнца из окна видны пылинки:

```tsx
<motion.circle

  cy={randomInBeam()}

```

### 8.4 Блики от ламп

Вечером (`preset === 'evening'` или `night`):
- Жёлтые пятна под лампами (radial gradient)

```tsx
useEffect(() => {
  const handleVisibility = () => {

    }
  };
  document.addEventListener('visibilitychange', handleVisibility);

```
