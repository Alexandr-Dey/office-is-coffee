## Как использовать

1. **После каждого чек-пойнта** в TASKS.md (P0, P1, P2, P3, P4, P5) — пройдись по соответствующему разделу здесь

4. **Тестируй на разных устройствах**: iPhone Safari, Android Chrome, десктоп Chrome

## Раздел 1: После P0 (Фундамент)

### 1.1 Авторизация Google

**Шаги**:
1. Открой https://office-is-coffee.vercel.app в **режиме инкогнито** (важно — иначе ты залогинен старым)
2. Должен увидеть экран с кнопкой "Войти через Google"
3. Нажми кнопку

**Проверка**:
- [ ] Открывается стандартный Google popup
- [ ] После выбора аккаунта popup закрывается
- [ ] Происходит редирект на /menu или /onboarding
- [ ] В Firestore → Console → users → видишь новый документ с твоим uid и role: 'client'
- [ ] Имя из Google аккаунта подтянулось в displayName

**Как проверить роль бариста/CEO**:
1. Запусти `scripts/set-roles.ts` (см. P0-1)
2. Выйди из приложения, войди снова
3. Тебя должно редиректнуть на /admin (для бариста) или /ceo

### 1.2 Cloud Functions

**Шаги**:
1. Открой Firebase Console → Functions
2. Должен увидеть 4-5 функций со статусом "Active"

**Проверка**:
- [ ] `onOrderCreate` — есть
- [ ] `onOrderReady` — есть
- [ ] `onDepositTopup` — есть
- [ ] `onCafeOpen` — есть
- [ ] `scheduledStreakCheck` — есть (либо закомментирована если на Spark плане)

**Тест функций**:
1. Создай тестовый заказ через приложение
2. В Firestore проверь `orders/{id}` — статус должен сразу стать `pending` (это сделала функция)
3. Через админку (с ролью бариста) переведи в `accepted`, потом в `ready`
4. Проверь `barista_bonuses/{твой_uid}` — должен появиться бонус +5₸

### 1.3 PWA иконки

**Тест на iPhone**:
1. Открой сайт в Safari
2. Нажми иконку "Поделиться" → "На экран Домой"
3. В превью должна быть **красное сердце Love is Coffee** (не белый квадрат)
4. Добавь на экран
5. Иконка на главном экране — красное сердце
6. Тапни → открывается приложение в standalone режиме (без адресной строки)

**Тест на Android**:
1. Открой сайт в Chrome
2. В адресной строке должна появиться кнопка "Установить"
3. Нажми → видишь иконку и название
4. После установки — иконка на главном экране

**Проверка**:
- [ ] iOS: иконка корректная при добавлении на Home Screen
- [ ] iOS: открывается без адресной строки (standalone)
- [ ] Android: предложение установить появляется
- [ ] Android: иконка корректная

### 1.4 Push-уведомления

**Шаги**:
1. Открой /menu (после авторизации)
2. Должен появиться системный запрос "Разрешить уведомления?"
3. Разреши

**Проверка**:
- [ ] Запрос появляется
- [ ] После разрешения — в Firestore `push_tokens/{твой_uid}` появился документ с токеном
- [ ] В Firebase Console → Cloud Messaging → "Send test message" → ввести твой токен → отправить
- [ ] Уведомление приходит на устройство (на iOS только в PWA режиме!)
- [ ] Тап на уведомление открывает приложение

**iOS специфика**:
- Push на iPhone Safari **в обычном браузере не работает** — это ограничение Apple
- Работает **только в PWA режиме** (когда добавлен на Home Screen)

### 1.5 Sentry (если настроен)

**Шаги**:
1. Если P0-7 выполнен — должна быть тестовая кнопка где-то (или временная)
2. Нажми → принудительно вызывается ошибка
3. Открой Sentry dashboard → должно появиться событие

## Раздел 3: После P2 (Сцена Phase 1)

### 3.1 Новая компоновка сцены

**Шаги**:
1. Открой /menu
2. Сцена должна занимать **верхние 60% экрана**

**Проверка**:

- [ ] Над сценой нет UI элементов (чистый визуал)
- [ ] Hero-карточка под сценой "выглядывает" из-под нижнего края
- [ ] Меню скроллится под сценой
- [ ] Сцена не залазит на BottomNav

### 3.2 Новая компоновка зала

**Проверка визуально**:
- [ ] Левая 1/3 — красная стена с логотипом Love is Coffee
- [ ] Центр и право — светлая стена с меню-бордами
- [ ] Окно справа (видна улица)
- [ ] Зелёная стойка на всю ширину центральной зоны
- [ ] На стойке: холодильник, микроволновка, кофемашина, кофемолка, касса, зона выдачи, стопки стаканчиков

### 3.3 Баристы и их жизнь

**Шаги**:
1. Залогинься как клиент
2. На /menu смотри сцену 1-2 минуты

**Проверка**:
- [ ] Виталий стоит слева, Аслан справа
- [ ] Они узнаваемые: разные фартуки (синий vs зелёный), разные причёски
- [ ] Каждые 1-3 секунды один из них меняет действие
- [ ] Видны разные действия: Виталий протирает, Аслан жонглирует, и т.д.
- [ ] Действия плавные, не дёргаются
- [ ] Через 30-60 секунд видно совместное действие (если реализовано)

### 3.4 Реакция на статус заказа

**Шаги**:
1. Сделай тестовый заказ через приложение
2. Смотри сцену в режиме реального времени
3. Через админку (другой браузер) переключай статусы

**Проверка**:
- [ ] При создании заказа (`pending`): баристы поворачиваются к экрану, появляется облачко с заказом
- [ ] При `accepted`: Виталий идёт к кофемашине, пар усиливается, Аслан пишет на стаканчике
- [ ] При `ready`: Аслан машет рукой, появляется облачко "Готово!"
- [ ] При `paid`: сцена возвращается в idle через 2 секунды

### 3.5 Грустный Виталий

**Шаги** (требует доступ к Firestore):
1. В Firestore → users → твой документ → измени `streak: 0` и `lastOrderDate: '2026-04-01'` (несколько дней назад)
2. Перезагрузи /menu

**Проверка**:
- [ ] Виталий выглядит грустным
- [ ] Над головой облачко с "?"

**Шаги**:
1. Проверь актуальную погоду в Алматы

**Проверка**:
- [ ] Если ясно: окна светлые, видны лучи света

## Раздел 9: Когда что-то сломалось

### Что делать:

1. **Скриншот** или **запись экрана**
2. Найди **точные шаги** воспроизведения
3. Открой **DevTools / Web Inspector** и посмотри **Console** на ошибки
4. Запиши в `KNOWN-ISSUES.md`:
```markdown
## ISSUE-001: Описание

**Шаги**: 
1. Открыть /menu
2. ...

**Ожидание**: что должно было произойти
**Реальность**: что реально произошло
**Console errors**: текст ошибки если есть
**Скриншот**: ![](url)
```
5. Скинь Claude Code с просьбой исправить
